# CourseMind NAS 学习视频系统路线图

时间：2026-06-02

## 最终目标

CourseMind 最终要脱离 Codex，作为绿联 DXP4800 NAS 上的后台常驻服务独立运行。

用户只需要把视频上传或同步到 NAS 指定目录，系统自动完成：

1. 检测新视频并等待文件同步稳定。
2. 判断视频是否像课程或教学内容。
3. 在白名单、额度、时长和重复调用保护通过后，生成字幕、章节、重点、笔记和搜索索引。
4. 在网页端提供手机和电脑都能访问的学习播放器。
5. 支持同步字幕、点击字幕跳转、章节跳转、重点跳转和播放进度恢复。

## 核心原则

1. 自动扫描不等于自动调用真实 ASR。
2. 默认 `AUTO_PROCESS_NEW_VIDEO=false`，避免 NAS 里大量视频误烧额度。
3. 已有 `raw_transcript.json` 时默认复用，不重复调用百炼。
4. 字幕优先保持自然完整的一句话或完整语义句，不机械按固定字数切碎。
5. Codex 只负责开发、维护和排查，正式运行必须由 NAS 后台服务独立完成。

## 当前状态

已完成：

1. 基础后端、前端、数据库、队列和播放器 gating。
2. `mock`、`aliyun_dashscope` 等 ASR provider 架构。
3. DashScope `fun-asr-realtime` 60 秒真实样片闭环。
4. 原始 ASR 到 `clean_transcript.json`、`smart_subtitle.vtt/srt` 的处理链路。
5. 自然完整句优先的显示字幕切分。
6. 热词和轻量纠错配置基础。
7. `raw_transcript.json` 离线回放重建脚本。

仍需完成：

1. NAS Docker 实机部署验证。
2. 文件稳定性检测，避免半同步视频被处理。
3. 后台常驻服务重启恢复。
4. 课程/非课程视频识别。
5. 自动处理白名单、黑名单、时长限制和每日额度保护。
6. 长视频切片、断点续跑和失败重试。
7. 结构化 AI 章节、重点、笔记和搜索增强。

## 推荐阶段

### 阶段一：稳定字幕地基

目标是保证真实字幕质量和成本安全。

1. 保持 `raw_transcript.json` 原始结果。
2. 持续优化完整句字幕切分。
3. 完善热词和纠错配置。
4. 用离线回放脚本反复调字幕，不重复调用 ASR。

### 阶段二：NAS 可部署

目标是让系统在 DXP4800 Docker 环境里可运行。

1. 固定 `/videos`、`/processed`、`/data`、`/logs`、`/config`。
2. 验证 FFmpeg、SQLite、中文路径和挂载权限。
3. 验证手机和电脑浏览器访问网页端。
4. 验证 NAS 重启或 Docker 重启后仍可恢复服务。

### 阶段三：后台自动化

目标是让用户只管上传视频，系统自己判断和处理。

1. 新视频文件稳定后再入库。
2. 自动判断 `course`、`non_course`、`unknown`。
3. 非课程视频不调用真实 ASR。
4. 待确认视频进入网页端人工确认。
5. 自动处理必须经过白名单、额度、时长和重复处理保护。

### 阶段四：AI 学习播放器

目标是接近百度网盘类学习体验，但更适合个人 NAS。

1. 字幕同步滚动。
2. 点击字幕跳转。
3. 点击章节跳转。
4. 点击重点跳转。
5. 自动生成课程笔记。
6. 支持按字幕、章节、重点、术语和笔记搜索。
7. 支持手机和电脑继续上次播放位置。

## 最小上线标准

能部署

能受控处理
不乱烧钱
