## 结论

你的需求应拆成两层：

1. Win11 和 Ubuntu 都能响应微信指令。
2. 同一条微信指令只能被一个 worker 执行。

解决方式不是复制登录态，而是建立统一队列和原子认领机制。

## 去重验证

远端测试结果：

```text
CLAIM_WIN=1
CLAIM_UBUNTU=0
REMOTE_DUAL_RUNNER_CHECK_OK
```

含义：同一条测试指令被 `win11-api-runner` 认领后，`ubuntu-account-runner` 再认领得到 0 条，因此不会重复执行。

## 安全边界

不会复制：

- `auth.json`
- `cap_sid`
- `.sandbox-secrets`
- API Key
- Cookie
- TOTP secret
- ChatGPT/OpenAI 桌面登录态

允许同步：

- skills
- rules
- memories
- plugins
- vendor_imports
- `memories_1.sqlite`
- `goals_1.sqlite`
- 非敏感配置/索引文件
