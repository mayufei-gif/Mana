# AgentHub 主管调度流程

## 一、主管只做 5 件事

1. 收需求。
2. 拆任务。
3. 分配角色。
4. 验收结果。
5. 统一发布。

## 二、任务分配格式

```text
任务ID：
项目：
角色：
目标：
允许修改：
禁止修改：
验收标准：
输出路径：
```

## 三、子角色开工前必须检查

1. 是否是自己的任务。
2. 是否有允许修改文件。
3. 是否有 workspace。
4. 是否需要锁。
5. 是否碰到 token、Cloudflare、systemd、OpenClaw。

## 四、子角色交付后主管验收

主管检查：

1. 是否只改允许文件。
2. 是否有结果报告。
3. 是否有验证命令。
4. 是否有风险说明。
5. 是否需要同步到 Ubuntu。
6. 是否需要重启服务。

## 五、发布规则

只有主管对话可以统一执行或指导：

1. `scp` 同步。
2. `sudo systemctl restart ...`
3. Cloudflare 配置。
4. OpenClaw 容器补丁。

子角色不能自己发布生产。
